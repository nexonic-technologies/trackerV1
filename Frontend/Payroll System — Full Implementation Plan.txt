Backend — Services
payrollEngine.js (new service — the core business logic)

This is the most important new file. Responsibilities:

computeAttendanceSummary(employeeId, month, year) — queries attendances collection, counts:

presentDays (Present + WFH + Late Entry)

lopDays (LOP status)

leaveDays from approved leaves in that month

workingDays = calendar days − weekoffs − holidays

overtimeHours from days where workHours > shift.overtimeThreshold

computeSalary(employeeId, month, year, attendanceSummary) — reads SalaryStructure for the employee, then:

earnedBasic = (basic / workingDays) * presentDays

earnedHRA = (hra / workingDays) * presentDays (proportional)

lopDeduction = (basic / workingDays) * lopDays

pfDeduction = earnedBasic * pfEmployee% (capped at ₹15,000 PF ceiling)

esiDeduction = gross < 21000 ? gross * 0.0075 : 0 (Indian ESI rule)

tds = employee.salaryDetails.tdsMonthly or bracket-based estimate

grossSalary = earnedBasic + allowances + overtimePay

netSalary = grossSalary − pfDeduction − esiDeduction − tds − other

runPayrollForEmployee(employeeId, month, year, processedBy) — orchestrates 1+2, creates/updates the Payroll record with status Processed, links to PayrollRun

runBulkPayroll(month, year, employeeIds, initiatedBy) — creates a PayrollRun document, queues each employee via the existing Bull computationService queue (new job type: payroll-compute), updates PayrollRun.status to Processing → Completed

approvePayrollRun(runId, approvedBy) — transitions PayrollRun.status Draft→Approved, validates all records are Processed

markRunAsPaid(runId, paidBy) — bulk-updates all Payroll records in the run to Paid, stamps paidAt

Business Logic Rules (the engine decisions)
Scenario	Rule
Employee absent with no approved leave	Status = LOP; lopDays incremented; salary deducted proportionally
Approved leave (within leave balance)	No deduction; presentDays counted
LOP leave (leave balance exhausted)	Treated same as absent — proportional deduction
PF	Employee: 12% of basic (capped at ₹1,800/month if basic > ₹15,000). Employer: 12% of basic (stored separately in SalaryStructure, not deducted from net)
ESI	Applicable only if gross salary ≤ ₹21,000. Employee: 0.75%, Employer: 3.25%
TDS	Use flat monthly value from SalaryStructure.tdsMonthly. Future: compute from annual bracket
Overtime	overtimeHours * overtimeRate (rate from SalaryStructure or passed at run time)
Duplicate prevention	Payroll has unique index on {employeeId, month, year} — re-running replaces existing Draft/Processed record, blocked if status = Paid
Payroll Run state machine	Draft → Processing → Completed → Approved → Paid (no reverse)
Employee inactive/terminated	Excluded from bulk run unless explicitly included
Data Flow Summary
HR triggers bulk run (month/year)
  → POST /payroll/run
  → payrollEngine.runBulkPayroll()
  → creates PayrollRun (status: Processing)
  → queues N jobs in Bull (payroll-compute)
    → for each employee:
       computeAttendanceSummary() → reads attendances + leaves
       computeSalary()           → reads salarystructures
       upsert Payroll record     → status: Processed
  → PayrollRun status: Completed

HR reviews & approves
  → POST /payroll/approve/:runId
  → PayrollRun status: Approved, all Payroll records: Processed

HR marks as paid
  → POST /payroll/pay/:runId
  → PayrollRun status: Paid, all Payroll records: Paid, paidAt stamped

Employee reads own slip
  → GET /populate/read/payrolls (filter: employeeId=self, month, year)
  → Access policy: isSelf → allowed
  → Returns single record with salary breakdown
